Você é um engenheiro full-stack sênior. Quero que você crie um projeto de site inspirado no conceito do GG.deals, mas sem copiar identidade visual, marca, textos ou layout exato. O site será focado em comparação de preços de jogos digitais com ativação Global, América Latina e Brasil.

Nome provisório do projeto:
GameDeals BR

Objetivo do produto:
Criar uma plataforma onde o usuário possa pesquisar jogos digitais e comparar preços entre lojas, com foco em:
- Chaves com ativação Global
- Chaves com ativação América Latina / LATAM
- Chaves com ativação Brasil
- Preços em BRL, USD e outras moedas relevantes
- Histórico de menor preço
- Alertas de preço
- Lista de desejos
- Comparação entre lojas oficiais e marketplaces, deixando claro o tipo da loja
- Destaque para lojas confiáveis, região de ativação e possíveis restrições

Importante:
Não quero copiar o GG.deals diretamente. Quero um produto com proposta parecida, mas com identidade própria, focada no público brasileiro e latino-americano.

Stack sugerida:
- Frontend: Next.js com TypeScript
- Estilização: Tailwind CSS
- Backend/API: Next.js API Routes ou NestJS, se o projeto já usar backend separado
- Banco de dados: PostgreSQL
- ORM: Prisma
- Cache: Redis
- Jobs/agendamento: BullMQ, node-cron ou equivalente
- Autenticação: NextAuth/Auth.js
- Deploy: Vercel para frontend e backend simples; Railway/Supabase/Render para banco e workers

6. APIs externas de preço
Implementar camada de integração com múltiplas fontes:

Fonte 1: IsThereAnyDeal API
Usar para:
- Buscar jogos
- Obter overview de preços
- Obter menor preço atual
- Obter histórico de menor preço
- Obter lojas disponíveis
- Validar regiões quando disponível

Fonte 2: CheapShark API
Usar como fonte complementar para:
- Deals de jogos digitais
- Busca por título
- Comparação de preços em lojas suportadas

Fonte 3: Steam Store API / appdetails
Usar para:
- Buscar preço de jogos Steam por país
- Usar parâmetro de país, por exemplo cc=BR
- Obter preço em moeda local quando disponível
- Nunca consultar em tempo real a cada acesso do usuário; usar cache

Importante:
Não depender exclusivamente de scraping. Priorizar APIs oficiais ou públicas. Caso algum scraping seja necessário no futuro, criar um módulo separado respeitando robots.txt, termos de uso, rate limit e cache.

7. Atualização de preço
Criar uma lógica de atualização quase em tempo real usando cache e jobs:

- Não consultar APIs externas a cada carregamento de página.
- Criar job recorrente para atualizar preços:
  - Ofertas populares: a cada 15 ou 30 minutos
  - Jogos normais: a cada 3 a 6 horas
  - Jogos pouco acessados: a cada 12 ou 24 horas
- Quando o usuário abrir uma página de jogo:
  - Mostrar dados do cache imediatamente
  - Se lastCheckedAt for antigo, disparar atualização assíncrona
  - Exibir "Atualizado há X minutos"
- Salvar todo preço consultado em PriceHistory
- Usar Redis para cache de:
  - Busca
  - Página do jogo
  - Lista de ofertas
  - Conversão de moedas
